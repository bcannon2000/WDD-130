What are Chiplets?

First, we need to talk about what a normal chip or dies are. The most common chips, in most consumer devices, are called monolithic or a single piece of silicon. When a chip is broken into separate parts, there are multiple levels of disaggregation. The most common version is called chiplets, but there is a more "basic" version called MCM or Multiple Chip Module.

MCM is generally accepted as having multiple of the same dies interconnected to act as a single processor. This picture below shows a server CPU in both a hypothetical, large monolithic die compared to four of the same dies interconnected to each other.

Chiplets are individual functional units or components of an integrated circuit (IC) that are designed and fabricated separately and then interconnected to form a larger system. Instead of building a single monolithic IC containing all the required functions, chiplets allow for modular and flexible integration of different components.

How I view the difference between MCM and chiplets is based on how AMD explains it. The simplified version is that MCM is when multiple of the same die are interconnected but isn't dependent on having multiple chips. Chiplets are similar to MCM, but they use multiple different dies and generally rely on having one of each kind. For examples, click here. Why do we use Chiplets?

Each chiplet is typically designed to perform a specific function, such as a processor core, memory controller, input/output interface, or specialized accelerator. They are fabricated on separate dies using different semiconductor process technologies optimized for their specific functions. This modular approach allows for more efficient development and manufacturing as well as better customization and scalability.

Chiplets are interconnected using advanced packaging technologies, such as silicon interposers or through-silicon vias (TSVs), which provide the electrical and mechanical connections between the different chiplets. These interconnects enable high-speed data transfer and communication between the chiplets, forming a cohesive system.

  • Design flexibility: Different chiplets can be mixed and matched to create customized ICs tailored for specific applications, allowing for faster design iterations and shorter time-to-market.
  • Performance optimization: Each chiplet can be optimized for its specific function, allowing for better performance and power efficiency compared to a monolithic IC where all functions are integrated on a single die.
  • Technology scaling: Different chiplets can be manufactured using different process technologies, enabling the use of advanced nodes for certain functions while still using mature or specialized technologies for others.
  • Yield improvement: Like all technology, there is a given chance of errors. Generally, the error chance is around 0.2 to 0.08 defects per square cm. The bigger the die size, but more likely there will be an error that requires the die to be sold for cheaper and have a few parts disabled.
