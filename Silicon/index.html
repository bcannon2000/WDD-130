What are Chiplets?

First, we must talk about what a typical chip or dies are. The most common chips in consumer devices are monolithic or a single piece of silicon. When a chip is broken into separate parts, there are multiple levels of disaggregation. The most common version is called chiplets, but a more "basic" version of chiplet technology is called MCM or Multiple Chip Module.

MCM is generally accepted as having interconnected multiple of the same dies to act as a single processor. The picture below shows a server CPU in a hypothetical, large monolithic die compared to four of the same dies interconnected.

Chiplets are individual functional units or components of an integrated circuit (IC) designed, fabricated separately, and then interconnected to form a more extensive chip. Instead of building a single monolithic IC containing all the required functions, chiplets allow for modular and flexible integration of different components.

I view the difference between MCM and chiplets based on how AMD explains it. The simplified version is that MCM is when multiple of the same die are interconnected but are not dependent on having various chips. Chiplets are similar to MCM but use many different dies and generally rely on having one of each kind. For examples, click here. Below is a picture of the evolution of AMD's server processors.

Why do we use Chiplets?

Each chiplet is typically designed to perform a specific function, such as a processor core, memory controller, input/output interface, or specialized accelerator. They are fabricated on separate dies using different semiconductor process technologies optimized for their specific functions. This modular approach allows for more efficient development and manufacturing as well as better customization and scalability.

Chiplets are interconnected using advanced packaging technologies, such as silicon interposers or through-silicon vias (TSVs), providing electrical and mechanical connections between the different chiplets. These interconnect allow for high-speed data transfer and communication between the chiplets, forming a cohesive system.

  • Design flexibility: Different chiplets can be mixed and matched to create customized ICs tailored for specific applications, allowing for faster design iterations and shorter time-to-market.
  • Performance optimization: Each chiplet is optimized for its specific function. It can improve performance and power efficiency than a monolithic IC, where all parts are integrated on a single die.
  • Technology scaling: Different chiplets can be manufactured using different process technologies, enabling the use of advanced nodes for specific functions while still using mature or specialized technologies for others.
  • Yield improvement: Like all technology, there is a given chance of errors. Generally, the error chance is around 0.2 to 0.08 defects per square cm. The bigger the die size, but more likely, an error will require the die to be sold for cheaper and has a few parts disabled.
